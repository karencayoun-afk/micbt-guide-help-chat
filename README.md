# MiCBT Guide — "Lumi" AI Help (GitHub → Netlify)

A curated FAQ assistant for the MiCBT Guide app. It uses **semantic (embedding-based)
retrieval** over a hand-curated knowledge base, then answers with Claude — so every
reply is grounded in vetted MiCBT content you control.

## What's in here

```
index.html                      The Lumi chat UI (React, self-contained)
lumi.png                        Mascot
micbt_faq_database.xml          THE knowledge base — 202 curated FAQs (edit this to change answers)
embed.js                        Build script: turns the XML into faq_vectors.json (run locally)
faq_vectors.json                Generated by embed.js — commit it (NOT included yet; see step 2)
netlify.toml                    Netlify config (bundles the data files with the function)
.netlify/functions/chat.js      The backend: query embedding → cosine match → Claude
```

### The knowledge base (106 FAQs)
- **71 verbatim FAQs from the book** (weeks 1-9) — the exact Q&As written by Bruno & Andrea for
  weeks 1–9 (`source="book"`). Authoritative clinical wording, not paraphrased.
- **83 verbatim app & assessment docs** from the live BetterDocs site export
  (`source="app-help"`) — Get Started, Using the App, Account & Privacy, Troubleshooting,
  My Goals, My Assessments, all 10 Stage Guides, and the full assessment set:
  DASS-21, ES-16, MSES-R, MISS, IAI, SUDS, SWLS, and the Ethical Conduct Diary
  (definitions, how-to-use, and score interpretation).
- **33 curated special-population & integration FAQs** (`source="betterdocs"`) — trauma/PTSD,
  addiction, caregivers, aging, life-circumstance adaptation, and integration with other
  therapies. These don't exist in the book or the app docs.

## How retrieval works
1. `embed.js` (run once, locally) embeds every FAQ → `faq_vectors.json`.
2. At runtime, `chat.js` embeds only the user's question, compares it to the stored
   vectors by cosine similarity, takes the top 6 FAQs, and feeds them to Claude.
3. No vector database to host — the vectors live in the repo. If embeddings are
   unavailable, `chat.js` automatically falls back to keyword search.

## Setup (do once)

**1. Set environment variables** in Netlify → Site settings → Environment variables:
- `CLAUDE_API_KEY` — your Anthropic key (used to write answers)
- `EMBEDDING_API_KEY` — your Voyage AI key (used to embed questions)

**2. Generate the vectors and commit them:**
```bash
export EMBEDDING_API_KEY="pa-..."     # same Voyage AI key
node embed.js                         # writes faq_vectors.json (~200 vectors, small)
git add faq_vectors.json
git commit -m "Add FAQ vectors"
git push                              # Netlify auto-deploys
```

**3. Done.** Visit your Netlify URL. The chat will use semantic retrieval.

## Updating the FAQs
Edit `micbt_faq_database.xml` (add/edit `<faq>` entries), then re-run `node embed.js`
and commit the regenerated `faq_vectors.json`. The vectors must always match the XML.

## Embedding provider
Uses **Voyage AI `voyage-4`** (512 dims) — pairs naturally with your Anthropic/Claude
key and includes a generous free tier (200M tokens). Get a key at
https://dashboard.voyageai.com → API Keys (key starts with `pa-`).
To switch providers later, change the API call in `embedBatch()` (embed.js) and
`embedQuery()` (chat.js), and keep the model/dimension constants identical in both
files. That's the only contract — same model and dimensions in both, or query vectors
won't match the stored ones.

## Notes / known gaps
- **Rate limit** (6 questions/day per IP) is in-memory and resets when the function
  cold-starts — fine for testing, not durable. For production use Netlify Blobs or an
  external KV store.
- `index.html` is unchanged from your working version; the retrieval upgrade is entirely
  server-side.
